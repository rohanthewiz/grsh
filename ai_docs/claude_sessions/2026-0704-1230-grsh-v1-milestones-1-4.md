## Decisions made with the user (binding for future sessions)

1. **Syntax: hybrid, shell-first.** Bare lines are commands (bash-like);
   logic is Go. Chosen over Go-first and pure-library approaches.
2. **Engine: custom tree-walking interpreter** over `go/parser`/`go/ast`.
   yaegi rejected (stalled at Go 1.22, no module support); codegen+`go run`
   rejected (latency, toolchain dependency).
3. **v1 = script runner first**; REPL is v2 via the `runner.Session.Eval`
   seam (classifier scope, interp globals, side table all persist across
   Eval calls already).
4. **Script extension: `.grsh`** (user corrected from `.gsh` mid-plan).
5. Use the user's own libs throughout: `rohanthewiz/serr` v1.4.0 (error
   wrapping w/ key-val fields), `rohanthewiz/logger` v1.3.0 (`--debug`).

## Architecture (as built)

```
.grsh source
  → classify    (internal/classify)  per logical line: SHELL or GO; scope tracker
  → shellparse  (internal/shellparse) shell fragments → AST side table
  → transform   (internal/transform) line-preserving rewrite → valid Go file
                 header + `//line name:1` ⇒ all token.Pos report .grsh coords
  → go/parser   → interp (internal/interp, tree-walker, Value=any boxed natives)
  → shellexec   (internal/shellexec) pipelines/redirs/builtins via os/exec
```

Key mechanics:
- Shell chunk → `__shell(n)`; `$(...)` inside Go lines → `__capture(n)`;
  `n` indexes the side table of pre-parsed shell ASTs.
- Top-level `func name(...)` → `name := func(...)` (same line) + hoisting in
  `interp.Run` ⇒ forward refs and recursion work.
- `import "x"` → `__import("x")` (registry validation only; all pkgs preloaded).
- Classifier decision order (rules 1–7) in `internal/classify/classify.go`
  doc comment; `go` keyword deliberately classifies SHELL (`go build`).
  Notable fix: rule-6a dot only counts when selector-shaped ⇒ `cd ..` stays
  shell while `x.field` is Go.
- Classifier is seeded with builtin names (glob, status, errexit...) via
  `Classifier.Predeclare` — without this `errexit(true)` was shell.
- `fmt.Println/Print/Printf` are **stdio-bound** registry symbols
  (`stdlibreg.Package.Bound`) so Go-side output follows session stdio
  (goldens, future redirection). Other registry funcs are plain values
  called through one reflect boundary (`interp.callReflect`).
- `$(...)` yields (out, err): 1-LHS assign drops err silently (check
  `status()`); 2-LHS gets both. Generic funcs returning (T, error) with
  1 LHS abort on non-nil error. `errexit` failure = silent `ExitErr{status}`
  (bash set -e semantics).
- Word expansion deviations (documented in README): `$VAR` never splits
  (zsh-like); unquoted `$(cmd)` splits; `"$@"` always splices per-arg;
  `{expr}` string = one word, `[]string` splices.

## Package map

- `cmd/grsh` — flags: `-c`, `--debug` (serr chain via logger), `--explain`
  (prints per-line classification + rule), `--version`. Exit: 1 parse,
  2 runtime, script's own exit code otherwise.
- `internal/scan` — M1-era logical-line splitter; still used by nothing
  post-M2 except its tests (classify has its own joining). Candidate for
  removal/merge in cleanup.
- `internal/classify` — classifier + golines.go (go/scanner-driven
  continuation: semicolon-insertion rule, block-vs-composite-literal `{`).
- `internal/shellparse` — hand-rolled parser: CmdList/AndOr/Pipeline/
  Command/Word/Segments (Lit, EnvVar, CmdSub, GoExpr).
- `internal/shellexec` — State (real process env + chdir by design),
  expansion (field building w/ glob-escaped pattern), pipelines with
  os.Pipe, redirs, builtins (cd, export, unset, exit, alias, unalias,
  source, command). User-level failures print + status (127/126/1);
  hard errors only for exit/internal.
- `internal/transform` — chunk rewriting, `__capture` scan (skips Go
  strings/comments), import rewriting, top-level func rewrite.
- `internal/interp` — env.go (pointer-cell scopes), interp.go (stmts,
  frames+defer), expr.go (binary ops int/float64/string, assign rules,
  range incl. sorted string-key maps, switch, comma-ok), call.go
  (closures, callReflect, conversions, make/append/len/..., type asserts),
  structs.go (StructType/StructVal — script structs, dynamic fields).
- `internal/builtins` — glob, lines, fields, trim, readFile, writeFile,
  appendFile, exists, env, setenv, cd, pwd, args, status, ok, errexit,
  sh(cmdline), capture(cmdline).
- `internal/stdlibreg` — one file per pkg: fmt (Bound), strings, strconv,
  os, filepath, time, regexp, json (**adapted**: Parse/Marshal return
  string/any — no pointers in scripts), sort, math, errors, serr, logger.
- `internal/runner` — Session facade; ParseError type; UserMessage()
  renders `file:line: root cause`.
- `testdata/scripts/` — golden e2e: `*.grsh` + `*.want` (+ optional
  `.exit`, `.args`); `go test ./internal/runner -update` regenerates.
