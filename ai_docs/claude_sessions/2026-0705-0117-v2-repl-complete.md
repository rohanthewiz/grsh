## What was built

### internal/classify — REPL seams (`repl.go`)

- `ErrIncomplete` sentinel (was a string error in consumeGo). serr
  unwraps, so `errors.Is` works through the File() wrap.
- `Classifier.Clone()` — deep-copies the scope chain + depth; shares
  `pkgs` (read-only). Enables speculative classification.
- `Classifier.NeedsMore(src) bool` — runs `File` on a clone; true when:
  ErrIncomplete (mid-expression Go, open composite literal), clone depth
  > 0 after (open `{` block), or last non-blank chunk is shell with a
  trailing continuation (`\`, `|`, `&&`, `||`). Does NOT treat unbalanced
  shell quotes as continuation — matches script pipeline (shellparse
  errors at eval), documented in LANGUAGE.md.
- `Classifier.Names()` — all scope idents + registry package names,
  sorted, for completion.

### internal/runner

`Session.NeedsMore(src)` and `Session.Idents()` — thin delegates.

### internal/repl (new package)

- **chzyer/readline v1.5.1** chosen over bubbline: adds no deps beyond
  x/sys (already indirect) + x/term; bubbline would drag bubbletea tree
  (user prefers low-dependency).
- `Run(sess, version) int` builds readline, then `loop(sess, rd, errW)`
  — loop takes a `lineReader` interface so tests drive it with a fake.
- Loop: accumulate lines in buf → join → `NeedsMore`? keep reading with
  `  ... ` prompt : `Eval` the unit. ExitErr → return code (covers both
  `exit n` and errexit tripping = set -e exits the shell). Other errors
  print `grsh: <msg>` and continue.
- Prompt: `grsh <cwd~>> `, nonzero last status shown as `[N]`:
  `grsh ~/x [1]> `.
- Errors: `userMsg` strips the `<eval>:1:col:` prefix on single-line
  input; multi-line rewrites to `line N: ...` (regex `evalLoc`).
- Ctrl+C at prompt (readline.ErrInterrupt) drops pending buffer; Ctrl+D
  empty → exit with LastStatus, mid-continuation → abandon unit.
- **Signals**: readline is raw only inside Readline (verified
  operation.go:382 EnterRawMode/defer Exit), so terminal is cooked while
  children run → Ctrl+C SIGINTs the whole foreground group. REPL does
  signal.Notify(os.Interrupt) + drain goroutine for its lifetime: child
  dies, shell survives. Real job control deferred to the jobs milestone.
- History: `~/.grsh_history` via readline's HistoryFile. Limitation
  (deliberate): history is per physical line, not per multi-line unit —
  readline's file format can't hold newlines; joining with `;` would
  change classification on recall.
- Completer (`completer.go`, implements readline.AutoCompleter.Do):
  path-shaped word (`/`, leading `.` or `~`) → file completion (dirs get
  trailing `/`, `~` expanded for lookup only); command position (line
  start or after `| && || ; $(`) → PATH executables (scanned once,
  sync.Once) + live idents + Go keywords; else files + idents. Do()
  returns suffix-after-prefix + prefix length (chzyer convention).

### cmd/grsh

- No args + stdin TTY (x/term.IsTerminal) → `repl.Run`, exit its code.
- No args + piped stdin → ReadAll and `RunSource("<stdin>", ...)` —
  `echo 'ls' | grsh` works; empty stdin exits 0 (bash-like). This
  REPLACED the old "no args → usage + exit 2" contract; main_test
  updated accordingly (runIn helper feeds stdin).

## Gotchas learned

- chzyer/readline auto-saves each physical line to history; Config has
  DisableAutoSaveHistory if unit-level history is ever wanted.
- readline eats input buffered before the first Readline call — pty
  tests must pace their writes.
- `<eval>` error locations carry a column (`<eval>:1:0:`), so prefix
  stripping needs the regex, not TrimPrefix.
